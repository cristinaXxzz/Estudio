# Estudio · 对话式复习

一个陪你**边聊边复习**的网页：你把课件、笔记传进去，AI 把它们拆成一条条知识点，然后像一个聊得来的同学一样跟你聊——抛问题、追问、纠错、适度抬杠。掌握好的知识点以后少出现，掌握差的明天就会再来找你。

所有数据都存在**你自己的浏览器里**（像记在自己的本子上，不经过别人的服务器），可以一键备份到你的 GitHub 私有仓库当云端存档。

---

## 一、怎么启动

**方法一（最简单）：** 双击文件夹里的 **`启动.bat`**，浏览器会自动打开。

**方法二（命令行）：** 打开终端，输入这一条命令：

```
node D:\study\server.js
```

然后浏览器打开 **http://localhost:8460**

> 黑色窗口别关，关了网页就停了。需要先装过 Node.js（你电脑上已经有了）。

---

## 二、第一次用，按这个顺序来

1. **填 AI 接口**：第一次打开会自动弹出设置。把你的 AI 接口填进去：
   - 接口地址（比如 `https://api.xxx.com/v1`）
   - API Key（一串钥匙，`sk-` 开头的那种）
   - 模型名：点「拉取模型」，它会把这个接口支持的模型全列成下拉菜单，选一个就行（拉不到也可以手动输入）
   - 填完点「测试接口 1」，显示 ✓ 就行
   - **接口 2 可以不填**。填了的话：它会当"裁判"悄悄给你的回答打分、主接口挂了自动顶上、拆资料时两个接口一起干活（快一倍），还能解锁「三人群聊」模式
2. **建科目**：左边点「＋ 新科目」，起个名字（比如"微观经济学"）
3. **传资料**：点「＋ 上传文件」，把课件、笔记传进去（支持 txt / md / pdf / Word / **PPT / Excel / 网页 / csv**）。AI 会自动拆成知识点，稍等一会
4. **开始复习**：两种玩法——
   - 「💬 开始复习」：一对一，一个学伴陪你聊。它先跟你聊两句再自然进入正题，你用自己的话回答就行，也可以反问它、跟它抬杠、跑题——它会把话拉回来
   - 「👥 拉群复习」：弹出选人名单，勾几个 AI 同学拉群一起聊。自带两个角色：**小言**（引导推进）和**老杠**（挑刺抬杠举反例）；你也可以点右上角「🎭 角色」**自己造角色**——起名、写人设、选用哪个接口，造完就能拉进群。群里谁没话说谁自动闭嘴

## 它会记住你（学习记忆）

每次复习结束，AI 会自动提炼"这次了解到了用户什么"，存成学习记忆，分六类：**错题本**（薄弱点）、**迷雾区**（易混淆）、**高光墙**（掌握扎实的）、**方法柜**（哪种讲法对你管用）、**习惯档案**（你的学习偏好）、**待办窗台**（约好下次看的）。下次复习时它会自动带着这些记忆来——越用越懂你。记错了的可以在科目页"它记住的你"里手动删。

想让记忆检索更聪明，可以在设置里填一个**向量接口**（Embedding，把句子变成"数字指纹"按意思找相似的那种技术）。硅基流动（siliconflow.cn）注册就送额度，向量模型基本免费。**不填也完全能用**，只是检索方式从"按意思找"退化成"按字面找"。
5. **结束**：聊够了点右上角「今天到这 ✓」，它会自然收尾，这次的掌握情况自动记下来

## 三、怎么换 API Key

点右上角 **⚙️ 设置** → 把新的钥匙粘贴进对应的框 → 点「保存设置」。完事。

## 四、云备份（GitHub）

1. 打开 https://github.com/settings/tokens → 「Generate new token (classic)」
2. 勾上 **repo** 这一项权限，生成，把那串 `ghp_` 开头的字复制下来
3. 粘到设置里的「GitHub Token」框，保存
4. 之后每次复习结束会**自动备份**；也可以随时点「立即备份」
5. 换电脑/清了浏览器？在新机器上填同一个 Token，点「从云端恢复」就全回来了

备份存在你账号下一个叫 `estudio-backup` 的**私有**仓库里（自动创建），别人看不到。
「走中转站连 GitHub」这个勾：在国内就保持勾上（直连 GitHub 经常不通）；在国外可以取消。

## 五、出错了？先试这三招

1. **刷新网页**（F5）。十有八九能解决。
2. **测试接口**：⚙️ 设置 → 点「测试接口 1」。不通的话多半是钥匙过期 / 余额用完 / 地址填错，换一个能用的就好。
3. **重启**：把黑色窗口关掉，重新双击 `启动.bat`。

还不行的话，把网页上红色提示的原话记下来发给帮你装这个的人（或者发给 AI 问）。**你的数据都在浏览器里，重启/报错不会丢。**

## 六、部署到 Vercel（手机电脑都能用）

全程点鼠标，不用写代码：

1. 打开 https://vercel.com → 「Sign Up」→ 选 **Continue with GitHub** 用你的 GitHub 账号登录
2. 进入控制台点 **Add New… → Project**
3. 在仓库列表里找到 **Estudio**，点它旁边的 **Import**（第一次会让你授权 Vercel 访问 GitHub，按提示点同意）
4. 配置页什么都不用改（Framework 显示 Other 就对了），直接点 **Deploy**
5. 等十几秒，出现彩带 🎉 就好了，你会得到一个网址，类似 `estudio-xxx.vercel.app`

之后每次这个仓库有更新，Vercel 会**自动重新部署**，你不用管。

**手机上用：**
- 用手机浏览器打开你的 vercel.app 网址
- iPhone：Safari 打开 → 分享按钮 → 「添加到主屏幕」；安卓：Chrome 打开 → 菜单 → 「添加到主屏幕」
- 之后桌面上就有个 Estudio 图标，点开全屏运行，像个真 App

**手机和电脑数据怎么同步：** 数据是各存各的（存在各自浏览器里），靠 GitHub 备份当"中转仓库"搬运：
1. 两边都填**同一个** GitHub Token（设置里）
2. 换设备用之前：点「从云端恢复」拉最新数据
3. 用完：自动备份会把进度传上去（默认开着）

简单记成一句话：**坐下先恢复，起身已备份**。别两台设备同时复习，后备份的会盖掉先备份的。

提醒：vercel.app 的网址在中国大陆访问可能不稳定；在国外用没问题。

## 七、几个放心点

- 不联网也能打开网页看记录，但聊天、拆资料、备份需要联网
- **不要清浏览器数据**（清缓存没事，清"网站数据/Cookie"会把本地记录清掉——所以记得开备份）

### 给技术人看的备注

纯静态单页应用，无构建步骤、零 npm 依赖。数据层 IndexedDB；AI 调用为 OpenAI 兼容格式（双 API 槽位：主聊 + 评估/兜底/并行提取）；防错解析与重试逻辑移植自 [SULLYOS](https://github.com/cristinaXxzz/X) 的 `safeApi.ts`；GitHub 备份用 Contents API 写入私有仓库，国内经 SULLYOS 已部署的 Cloudflare Worker（`/github` 白名单透传）中转；间隔重复为固定档位（掌握度 0–5 → 0.5/1/2/4/7/15 天）。PDF/DOCX 解析用 CDN 按需加载 pdf.js / mammoth。
